    Regras de Negócio

    Banco de Dados

    Categorias

    1. Regras de Restrição
    2. Cada categoria deve possuir um nome único e descritivo.
    3. Categorias não podem ser removidas se estiverem associadas a algum artigo, formulário de artigo ou tema.

    Formulário Tema

    1. Regras de Restrição
    2. Os campos nome, email e tema são obrigatórios para cada sugestão.
    3. O id_categoria deve referenciar uma categoria existente.

    Formulário Artigo

    1. Regras de Restrição
    2. Os campos nome, email, título e conteúdo são obrigatórios.
    3. O id_categoria deve referenciar uma categoria existente.

    Artigos

    1. Regras de Restrição
    2. Cada artigo deve ter um título único e conteúdo associado a um autor identificado pelo id_usuario.
    3. Apenas usuários cadastrados podem submeter artigos.

    Usuários

    1. Regras de Restrição
    2. O email dos usuários deve ser único no banco de dados.
    3. Regras de Restrição
    4. A senha deve ser armazenada de forma segura, utilizando técnicas de hash e sal.

    API

    Categorias

    1. Regras de Operação
    2. Listar Categorias: Todas as categorias cadastradas devem ser retornadas, ordenadas por nome em ordem alfabética.

    Feed (Artigos)

    1. Regras de Restrição
    2. Adicionar Artigo: O título e o conteúdo do artigo são obrigatórios e devem conter no mínimo 10 caracteres.

    Formulários (Sugestões)

    1. Regras de Restrição
    2. Adicionar Tema: O nome, e-mail e descrição do tema são obrigatórios.

    Login e Cadastro

    1. Regras de Causa e Efeito
    2. Realizar Login: O sistema deve validar e-mails e senhas em conformidade com a autenticação segura.

    WebForms

    Páginas Públicas

    1. Regras de Diretriz
    2. About.aspx: Exibe informações institucionais do blog.
    3. Default.aspx: Apresenta um feed de artigos publicados, organizados por ordem cronológica.

    Páginas Logadas

    1. Regras de Diretriz
    2. Profile.aspx: Exibe as informações do usuário logado e suas publicações pessoais.
    3. MeusArtigos.aspx: Usuários podem criar novos artigos, editar ou excluir artigos existentes.

    Validações Gerais

    1. Regras de Restrição
    2. Todos os formulários devem apresentar validações client-side e server-side para garantir consistência e segurança.

    Interação com a API

    1. Regras de Organização
    2. Toda manipulação de dados deve ser feita por meio de chamadas à API Conecta Ciência.
