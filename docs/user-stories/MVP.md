# MVP

## Introdução

O MVP, ou Produto Mínimo Viável, é uma abordagem utilizada no desenvolvimento de produtos que busca entregar a menor versão funcional possível de algo novo, com o objetivo de gerar valor imediato ao usuário e coletar feedback de maneira rápida e eficiente. Essa estratégia permite que equipes validem ideias e ajustem o produto com base em dados reais, reduzindo o risco de investir tempo e recursos em funcionalidades que podem não ser prioritárias ou valiosas para os usuários. O MVP também favorece a interação direta com os usuários desde as fases iniciais do projeto, criando oportunidades para entender melhor as expectativas e necessidades do público-alvo.

## US 

As US (User Stories) são o núcleo do desenvolvimento incremental. Cada história captura uma funcionalidade específica desejada pelo cliente ou usuário, estruturada para ser simples e facilmente compreendida. Seguindo o formato:

``` "Como [papel], eu quero [ação] para [valor do negócio]." ```

## Backlog

O backlog é uma lista dinâmica e ordenada que organiza todos os itens necessários para o desenvolvimento do produto. Essa lista inclui Histórias de Usuário, tarefas técnicas, correções de defeitos e outros requisitos relevantes.

O backlog vincula as US ao seu estado atual, possibilitando a priorização e refinamento progressivo. O Product Owner, juntamente com a equipe, utiliza esse backlog como a única fonte de verdade para o trabalho necessário, guiando o progresso do produto com base nas demandas mais urgentes e valiosas.

No nosso escopo, foram selecionadas as histórias de usuário que representam as funcionalidades essenciais para atender aos objetivos iniciais do produto. A lista a seguir detalha cada história escolhida:

1. **[US01 - Sobre](US01-Sobre.md):**  
   Eu, como usuário, quero acessar uma página com informações sobre o site para compreender seus objetivos e propósito.

2. **[US02 - Feed](US02-Feed.md):**  
   Eu, como usuário, quero acessar o feed com a listagem de artigos publicados, organizados em ordem cronológica, para encontrar novos conteúdos facilmente.

3. **[US03 - Filtros](US03-Filtros.md):**  
   Eu, como usuário, quero filtrar os artigos no feed para localizar conteúdos de forma mais eficiente e direcionada.

4. **[US04 - Sugestão de Tema](US04-SugestaoTema.md):**  
   Eu, como usuário, quero sugerir temas para artigos, contribuindo para a diversificação e relevância dos conteúdos publicados.

5. **[US05 - Sugestão de Artigos](US05-SugestaoArtigo.md):**  
   Eu, como usuário, quero enviar sugestões de artigos para publicação, auxiliando na curadoria de conteúdos.

6. **[US06 - Cadastro](US06-Cadastro.md):**  
   Eu, como usuário, quero me cadastrar no site para poder interagir e contribuir com a plataforma.

7. **[US07 - Acesso](US07-Acesso.md):**  
   Eu, como usuário, quero fazer login no site para acessar funcionalidades exclusivas, como publicação de artigos.

8. **[US08 - Perfil](US08-Perfil.md):**  
   Eu, como usuário logado, quero acessar uma tela com meu perfil para visualizar minhas informações e contribuições.

9. **[US09 - Publicação de Artigos](US09-Publicacao.md):**  
   Eu, como usuário logado, quero publicar artigos no site para compartilhar conteúdos científicos com outros usuários.

10. **[US10 - Meus Artigos](US10-MeusArtigos.md):**  
   Eu, como usuário logado, quero acessar uma página com os artigos publicados por mim para gerenciar meu conteúdo.

11. **[US11 - Editar e Apagar Publicação](US11-EditarExcluir.md):**  
   Eu, como usuário logado, quero editar ou excluir artigos que publiquei, garantindo que meu conteúdo esteja sempre atualizado.

12. **[US12 - Curtir e Comentar em Posts](US12-CurtirComentar.md):**
    Eu, como usuário, quero poder curtir e comentar em publicações no feed para interagir com o conteúdo.  

13. **[US13 - Visualizar Perfil Público de Outros Usuários](US13-PerfilPublico.md):**
    Eu, como usuário, quero poder visualizar o perfil público de outros usuários para acompanhar suas informações e publicações, sem precisar estar logado na conta deles.

14. **[US14 - Recomendações de Posts Personalizadas](US14-RecomendacoesFeed.md):**
    Eu, como usuário, quero receber recomendações de posts no feed baseadas nos meus interesses e interações.  

15. **[US15 - Seguir Outros Usuários ](US15-SeguirUsuarios.md):**
    Eu, como usuário, quero poder seguir outros usuários para acompanhar suas postagens de maneira prática.  

16. **[US16 - Notificações de Novas Publicações de Usuários Seguidos](US16-Notificacoes.md):**
    Eu, como usuário, quero receber notificações de novas publicações feitas pelos usuários que eu sigo para me manter atualizado sobre seus conteúdos.
